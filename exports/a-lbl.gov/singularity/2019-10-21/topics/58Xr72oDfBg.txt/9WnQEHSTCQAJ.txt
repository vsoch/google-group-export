> proper, and I would suggest you create your image using singularity and t=

>
> git clone -b development=20
> https://www.github.com/singularityware/singularity.git
> cd singularity
> ./autogen.sh
> ./configure --prefix=3D/usr/local
> make
> sudo make install
>
> Then for your docker image, you could do any of the following
>
> *1. Create and import*
> singularity create --size 6000 evolinc.img
> singularity import docker://ubuntu:latest   # (replace your Docker image=
=20
> uri here)
>
> *2. Pull*
> singularity pull docker://ubuntu:latest
>
> *3. Bootstrap*
> bootstrap is the full, proper container creation, and using the=20
> development branch you can use different sections to add files, labels,=
=20
> etc. You would make a file called Singularity that has something like:
>
>>
>> Bootstrap: docker
>> From: ubuntu:latest
>> %post
>> apt-get update && apt-get install -y git
>> # put other setup and install here
>> %files
>> /path/on/host /path/in/container
>> %runscript
>> # This is what your container does when you execute it
>> =20
>> exec echo Hello "$@"
>
>
>
> Check out the docs to see all the options=20
> <https://github.com/singularityware/singularityware.github.io/blob/docs/2=
.3/pages/docs/user-docs/docs-bootstrap.md>!=20
> Please give those a try then we can look more closely at the bug.
