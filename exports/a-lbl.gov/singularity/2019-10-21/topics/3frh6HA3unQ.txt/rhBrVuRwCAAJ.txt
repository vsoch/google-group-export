b) xattr over tmpfs

For security reasons, temporary file systems have disabled extended=20
attributes, since this is a possible security flaw by allowing a user to=20
write on memory pages of other namespace processes.=20

the log show you:


*Did you enable extended attributes (user_xattr) on the underlying file=20
system?*

what it means, enable this kernel feature for tmpfs -> *not recommended * a=
=20
test 1. "edit the extended attributes in a temporary file system" : this=20
will show "No works".
$ singularity shell --writable-tmpfs mgmtd.sif

Singularity mgtd.sif:~/> touch xattr_file && setfattr -n 'user.testAttr' -v=
 'attribute=20
value' xattr_file &> /dev/null; getfattr xattr_file 2>&1 | grep -q user.tes=
tAttr=20
&& echo 'It works!' || echo 'No works!'; rm xattr_file &> /dev/null



a test 2. "edit the extended attributes in a file system writable on disk"=
=20
: this will show "It works".=20
$ singularity shell mgmtd.sif

Singularity mgtd.sif:~/> touch xattr_file && setfattr -n 'user.testAttr' -v=
 'attribute=20
value' xattr_file &> /dev/null; getfattr xattr_file 2>&1 | grep -q user.tes=
tAttr=20
&& echo 'It works!' || echo 'No works!'; rm xattr_file &> /dev/null
