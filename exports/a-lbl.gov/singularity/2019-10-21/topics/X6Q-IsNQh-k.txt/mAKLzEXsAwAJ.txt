Little bit more info after some digging...

SIngularity 3.x now uses the OCI containers/image library to handle 
fetching of docker and other oci containers in a more robust and compliant 
way than our own code in 2.x did. The error is happening in that library as 
it tries to read authentication credentials that might have be configures 
using podman-login at `$XDG_RUNTIME_DIR/containers/auth.json` 

It seems the check for that isn't handling the case that $XDG_RUNTIME_DIR 
doesn't exist... which is what you can have in an SGE batch job vs an 
interactive session.

Ref: 
https://github.com/containers/image/blob/8ad47b442feae0a21bff51a6058227fa1e45bb41/docs/containers-transports.5.md#dockerdocker-reference

Would you mind opening an issue at: 
https://github.com/sylabs/singularity/issues ?

Cheers,
